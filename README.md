# 双图隐写

一个离线单页 Web 工具，用两张图片共同隐藏和读取一段短文字。适合把名字缩写、日期、短暗号这类内容藏进两张头像里：单张图片无法直接读出内容，必须两张一起读取才会还原文字。

## 文件

- `double-image-secret.html`：主工具，直接用浏览器打开即可使用。
- `output/verification/`：开发验证时生成的截图和测试图片，可忽略。

## 使用方法

### 写入

1. 打开 `double-image-secret.html`。
2. 进入 `写入`。
3. 上传两张图片。
4. 输入不超过 40 字节的短文字，例如 `AZ&LY 0521/1018`。
5. 点击 `生成图片`。
6. 同时保存生成的两张图片，优先使用 PNG。

### 读取

1. 进入 `读取`。
2. 上传写入后生成的两张图片。
3. 点击 `读取文字`。
4. 工具会显示算法流程、播放双图汇合动效，并突出显示解密文字。

## 原理简述

工具会把短文字编码成 payload，并加入版本、长度和 CRC32 校验。随后使用 XOR 双份额拆分：一份写入图片 A，另一份写入图片 B。读取时必须同时提取两份并 XOR 合并，才能还原原始文字。

隐写部分使用头像归一化、Hamming 纠错、重复映射和 DCT 中频系数调整，让普通肉眼看不出明显变化，并尽量提高对头像压缩和普通头像裁剪的容忍度。

## 注意事项

- 全部处理都在浏览器本地完成，不会上传图片。
- 推荐保存 PNG，高质量 JPG 通常也可读取。
- 普通头像平台的中心裁剪、轻微偏移裁剪会自动尝试读取；重裁剪、滤镜、强压缩、重新截图等操作仍可能导致读取失败。
- 目标是日常不显眼，不是专业取证级隐写。
- 不设置密码时，任何人只要拿到两张图和这个工具，都可能读取出隐藏文字。

- PNG 写入和读取正常。
- 高质量 JPG 写入和读取正常。

- 旧版 `VERSION=1` 的测试图片仍可读取。
- 桌面和移动端布局已检查。
